# kud.io — project instructions

## Language: British English

All **user-facing copy on this site uses British English** (e.g. _colour_,
_organise_, _behaviour_, _licence_ as a noun, _customise_, _centre_). This
applies to everything authored in this repo that a visitor reads:

- Page text, headings, and intros (`app/**`)
- `metadata` titles/descriptions and OpenGraph copy
- Category names and blurbs in `app/projects/page.tsx`
- Component copy, `alt` text, `aria-label`s, and empty/error states

**Exceptions — keep the original (American) spelling:**

- Code identifiers and CSS properties: `color`, `background-color`, the JS DOM
  API (`behavior: "smooth"`, `alignItems: "center"`, `scrollBehavior`).
- Field names from third-party APIs and their data (e.g. Raycast's
  `download_count`, GitHub fields) — match the upstream name in code.

**Out of scope — do not "correct" these:**

- `content/readmes/**` and `content/projects/**/docs/**` are **synced from the
  source repos** (the single source of truth) and regenerated by
  `scripts/sync-content.js`. Never hand-edit spelling there; fix it upstream in
  the source repo's README/docs instead. See `REPO-CONVENTIONS.md`.
