<Section
  eyebrow="Why shui"
  title="Stop hand-rolling ANSI escapes"
  sub="Most Zsh scripts scatter raw escape codes everywhere — unreadable, inconsistent, impossible to theme. shui gives you semantic components and a token-based theme engine instead."
>
  <Install label="without shui">

```zsh
echo -e "\033[32m✓\033[0m Deployed"
echo -e "\033[1;31m✗\033[0m Build failed"
echo -e "\033[33m⚠\033[0m  Low disk space"
```

  </Install>

  <Install label="with shui">

```zsh
shui message success "Deployed"
shui message error   "Build failed"
shui message warning "Low disk space"
```

  </Install>
</Section>

<Section eyebrow="Quick start" title="Up and running in seconds">
  <Install label="zsh">

```sh
git clone https://github.com/kud/shui ~/.shui
source ~/.shui/shui.zsh

shui message success "Deployment complete"
```

  </Install>
</Section>

<Section
  eyebrow="Features"
  title="Built for real shell scripts"
  sub="Everything you need to give your scripts a consistent, themeable voice."
>
  <Features>
    <Feature title="Unified message API">
      One consistent command for success, error, warning, info, and muted
      messages.
    </Feature>
    <Feature title="Token-based themes">
      Swap colours, icons, and styles via environment variables without touching
      component code.
    </Feature>
    <Feature title="Inline components">
      Badges and pills write to stdout without a newline, composing naturally
      inside command substitutions.
    </Feature>
    <Feature title="Interactive prompts">
      Confirm, select, radio, and input build polished interactive layers with
      sensible defaults.
    </Feature>
    <Feature title="Zero dependencies">
      A single source file, pure Zsh — no npm, no brew, nothing to install.
    </Feature>
    <Feature title="NO_COLOR aware">
      Respects the NO_COLOR standard and strips styling in constrained
      environments.
    </Feature>
  </Features>
</Section>
